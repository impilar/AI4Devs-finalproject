## Historia de usuario

> Como usuario final, quiero quitar una etiqueta de una nota sin borrar la nota, para reorganizar sin perder contenido.

## Criterios de aceptación (Gherkin)

```gherkin
Feature: Quitar etiqueta de nota

  Scenario: Desasociar etiqueta desde detalle o edición
    Given que la nota "Reunión" tiene las etiquetas "trabajo" y "urgente"
    When quito la etiqueta "urgente"
    Then la nota "Reunión" ya no muestra "urgente"
    And la nota sigue existiendo con el resto de sus datos

  Scenario: La etiqueta permanece si otras notas la usan
    Given que "trabajo" está en las notas "A" y "B"
    When quito "trabajo" solo de la nota "A"
    Then la etiqueta "trabajo" sigue existiendo
    And la nota "B" conserva la etiqueta "trabajo"
```

## Detalle de implementación (historia)

### Resumen técnico

El backend expone `DELETE /api/v1/notas/:notaId/etiquetas/:etiquetaId` que elimina **solo** la fila en `nota_etiqueta` mediante `NotaService.removeTag`. La nota y la etiqueta global permanecen; si otras notas usan la misma etiqueta, no se ve afectadas. El frontend añade botón «quitar» (X) en chips de etiqueta en `NoteDetail` (vista lectura) y/o en `TagInput` (modo edición), actualizando la UI tras éxito (`204`) sin recargar toda la página si es posible.

### API involucrada

| Método | Ruta | Uso en esta historia |
|--------|------|----------------------|
| DELETE | `/api/v1/notas/:id/etiquetas/:etiquetaId` | Desasociar una etiqueta de una nota |
| GET | `/api/v1/notas/:id` | Refrescar detalle tras quitar (opcional si UI optimista) |

**Respuesta 204:** sin cuerpo.

**Errores:**

| HTTP | code | Cuándo |
|------|------|--------|
| 404 | `NOT_FOUND` | Nota inexistente o asociación no existe |
| 400 | `VALIDATION_ERROR` | UUID inválido en params |

### Contratos (DTOs / campos)

| Parámetro | Tipo | Notas |
|-----------|------|-------|
| `id` (nota) | UUID | Path param |
| `etiquetaId` | UUID | ID de fila en `etiquetas`, no nombre |

El detalle `NotaDetail` sigue exponiendo `tags: string[]` (nombres); el FE resuelve `etiquetaId` desde detalle enriquecido o endpoint auxiliar. **Propuesta si falta ID en DTO:** extender `NotaDetail` con `tags: { id, name }[]` solo si necesario — marcar en implementación; alternativa: lookup por nombre vía `GET /etiquetas` existente.

### TASK-039 — [DB] DELETE en nota_etiqueta sin cascade a entidades

| Campo | Valor |
|-------|-------|
| **Tipo** | Database |
| **Release** | V1 |
| **Depende de** | US-008 TASK-031 (tabla `nota_etiqueta`) |

**Objetivo:** Verificar que eliminar una fila de `nota_etiqueta` no borra la nota ni la etiqueta maestra.

**Archivos a crear o modificar:**

- `src/backend/prisma/schema.prisma` — revisar FKs en `NotaEtiqueta`
- `src/backend/prisma/migrations/20260704130000_add_etiquetas_mvp/migration.sql` — DDL FK
- `src/backend/src/repositories/nota.repository.ts` — `deleteMany` solo en `notaEtiqueta`
- Test integración o script de verificación

**Pasos de implementación:**

1. Confirmar FK: `nota_etiqueta.nota_id` → `notas.id` ON DELETE CASCADE (borrar **nota** sí borra asociaciones; no al revés).
2. Confirmar FK: `nota_etiqueta.etiqueta_id` → `etiquetas.id` ON DELETE CASCADE aplica al borrar **etiqueta**, no al borrar fila puente.
3. Implementar `removeTag` con `prisma.notaEtiqueta.delete({ where: { notaId_etiquetaId: { notaId, etiquetaId } } })` — no `delete` en `etiqueta` ni `nota`.
4. Test: dos notas con misma etiqueta; quitar de una → fila `etiquetas` sigue existiendo; otra nota conserva asociación.

**Criterios de done (verificables):**

- [ ] DELETE asociación no elimina fila en `notas`
- [ ] DELETE asociación no elimina fila en `etiquetas`
- [ ] Segunda nota mantiene `nota_etiqueta` con misma `etiqueta_id`
- [ ] Sin migración nueva salvo corrección de FK incorrecta (documentar si aplica)

**Tests:**

| Tipo | Ubicación | Qué valida |
|------|-----------|------------|
| Integración | `tests/integration/api/notas.remove-tag.test.ts` | Persistencia nota y etiqueta global |

**Notas técnicas:** `ON DELETE CASCADE` en FK de `nota_etiqueta` hacia padres es correcto para borrado de nota/etiqueta entera; US-010 solo borra la fila puente.
