## Historia de usuario

> Como usuario final, quiero un mensaje claro cuando la búsqueda no devuelve resultados, para saber que debo probar otros términos.

## Criterios de aceptación (Gherkin)

```gherkin
Feature: Búsqueda sin resultados

  Scenario: Mensaje claro sin coincidencias
    Given que no existe ninguna nota con el término "xyzabc"
    When busco "xyzabc"
    Then veo el texto "Sin resultados para xyzabc"
    And no veo mensajes de error técnico

  Scenario: Campo de búsqueda permanece editable
    Given que una búsqueda no devolvió resultados
    When veo el mensaje de sin resultados
    Then el campo de búsqueda sigue visible y puedo modificar el término
```

## Detalle de implementación (historia)

### Resumen técnico

Cuando `GET /api/v1/buscar?q={term}` no encuentra coincidencias, el backend responde `200` con `data: []` y metadata que incluye el término buscado (`meta.q` en LLD actual; alias `searchTerm` aceptable si el FE lo prefiere). El frontend muestra `SearchEmptyState` con texto **«Sin resultados para {término}»** (literal Gherkin) y mantiene `SearchBar` visible y editable para reintentar. No mostrar `ErrorMessage` ni códigos HTTP de error en este caso.

### API involucrada

| Método | Ruta | Uso en esta historia |
|--------|------|----------------------|
| GET | `/api/v1/buscar?q={term}` | Búsqueda sin coincidencias → `200`, `data: []` |

**Respuesta 200 (sin resultados):**

```json
{
  "data": [],
  "meta": {
    "q": "xyzabc",
    "total": 0
  }
}
```

**Nota TASK-053:** El roadmap nombra `searchTerm`; el código MVP usa `meta.q`. V1 debe garantizar que el FE recibe el término en metadata (`q` o ambos `q` + `searchTerm`) para renderizar el mensaje. No cambiar a 404 ni 204.

### Contratos (DTOs / campos)

| Campo | Tipo | En sin resultados | Notas |
|-------|------|-------------------|-------|
| `data` | `NotaResumen[]` | `[]` | Sin error |
| `meta.q` | string | término normalizado (trim) | Eco del query param |
| `meta.total` | number | `0` | Coherente con `data.length` |

Query: `SearchQuerySchema` — `q` obligatorio, trim, min 1.
