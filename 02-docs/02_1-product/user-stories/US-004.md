## Detalle de implementación (historia)

### Resumen técnico

El usuario elige ordenación en un selector en `HomePage` (listado normal, sin búsqueda activa). `useNotes` llama `GET /api/v1/notas` con `sort=created_at|title` y `order=asc|desc`. El backend ya valida estos params con `ListNotasQuerySchema` y `nota.repository` aplica `orderBy`; V2+ completa cobertura de tests BE, expone el selector en FE y persiste la preferencia en `sessionStorage` para la sesión del navegador. La ordenación aplica al listado completo y al filtrado por etiqueta (`?etiqueta=`); no al flujo de búsqueda (`/buscar`, US-012/013).

### API involucrada

| Método | Ruta | Uso en esta historia |
|--------|------|----------------------|
| GET | `/api/v1/notas` | Listado con `sort` y `order` |
| GET | `/api/v1/notas?etiqueta={nombre}&sort=title&order=asc` | Ordenación combinada con filtro etiqueta |

**Query params:**

| Param | Tipo | Obligatorio | Default | Valores |
|-------|------|-------------|---------|---------|
| `sort` | enum | no | `created_at` | `created_at` \| `title` |
| `order` | enum | no | `desc` | `asc` \| `desc` |
| `etiqueta` | string | no | — | Compatible con US-009 |

**Respuesta 200:** sin cambios respecto a US-001 — `{ data: NotaResumen[], meta: { total } }` con el orden solicitado.

**Errores:** `400 VALIDATION_ERROR` si `sort` u `order` inválidos (ya implementado parcialmente).

### Contratos (DTOs / campos)

| Campo JSON | Tipo | Notas |
|------------|------|-------|
| `data[].title` | string | Orden alfabético cuando `sort=title` |
| `data[].createdAt` | ISO datetime | Orden cronológico cuando `sort=created_at` |
| `meta.total` | number | Invariante respecto al orden |
