# Glosario

| Término | Definición |
|---------|------------|
| **Nota** | Unidad principal de información: título, contenido, fechas y metadatos opcionales |
| **Etiqueta (Tag)** | Clasificador temático reutilizable; relación many-to-many con notas |
| **Enlace** | URL externa asociada a una nota |
| **MVP** | Minimum Viable Product: CRUD notas, etiquetas, búsqueda simple |
| **Backlink** | Referencia bidireccional entre dos notas (fuera del MVP) |
| **Grafo de conocimiento** | Visualización de relaciones entre notas (fuera del MVP) |
| **HLD** | High Level Design — arquitectura de sistema versionada en `02-docs/02_2-architecture/hld/` |
| **ADR** | Architecture Decision Record — decisión técnica documentada en `02-docs/02_2-architecture/adr/` |
| **PRD** | Product Requirements Document — requisitos de producto en `02-docs/02_1-product/prd/` |
| **User Story Map** | Mapa de historias de usuario (método Jeff Patton) |
| **INVEST** | Criterios para historias: Independent, Negotiable, Valuable, Estimable, Small, Testable |
