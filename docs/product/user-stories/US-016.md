## Detalle de implementación (historia)

### Resumen técnico

El backend expone `DELETE /api/v1/notas/:id` que elimina la nota de forma irreversible (sin soft-delete); PostgreSQL CASCADE borra enlaces y filas `nota_etiqueta` asociadas. El controller responde `204` sin cuerpo. El frontend muestra botón «Eliminar» en `NoteDetailPage` que abre `ConfirmDialog`; al confirmar, `useNote` llama la API y redirige a `HomePage`; al cancelar, la nota permanece intacta.

### API involucrada

| Método | Ruta | Uso en esta historia |
|--------|------|----------------------|
| DELETE | `/api/v1/notas/:id` | Eliminar nota permanentemente |
| GET | `/api/v1/notas/:id` | Verificar 404 tras eliminación (irreversible) |
| GET | `/api/v1/notas` | Verificar nota ausente del listado |

**Path param:** `id` — UUID de la nota.

**Respuesta éxito:** `204 No Content` (cuerpo vacío).

**Errores:**

| HTTP | code | Cuándo |
|------|------|--------|
| 404 | `NOT_FOUND` | Nota inexistente o ya eliminada |
| 400 | `VALIDATION_ERROR` | `id` no es UUID válido |

### Contratos (DTOs / campos)

## Detalle por task

### TASK-061 — [BE] Endpoint DELETE /notas/:id irreversible

| Campo | Valor |
|-------|-------|
| **Tipo** | Backend |
| **Release** | MVP |
| **Depende de** | TASK-005 (GET detalle US-002), TASK-063 (CASCADE en BD) |

**Objetivo:** Exponer eliminación permanente de nota bajo `DELETE /api/v1/notas/:id` con respuesta 204.

**Archivos a crear o modificar:**

- `src/backend/src/routes/notas.routes.ts` — `DELETE /:id` → `notaController.deleteNota`
- `src/backend/src/controllers/nota.controller.ts` — método `deleteNota`
- `src/backend/src/services/nota.service.ts` — método `delete(id)`
- `src/backend/src/repositories/nota.repository.ts` — método `delete(id)`
- `src/backend/src/schemas/common.schema.ts` — `IdParamSchema`

**Pasos de implementación:**

1. En `notas.routes.ts`, registrar `DELETE /:id` con `validateParams(IdParamSchema)`.
2. En `nota.controller.deleteNota` (LLD §5.3):
   - Llamar `notaService.delete(req.params.id)`.
   - Responder `204` sin cuerpo (`res.status(204).send()`).
3. En `nota.service.delete`:
   - `findById`; si null → `NotFoundError` 404.
   - `notaRepository.delete(id)` — CASCADE elimina `enlaces` y `nota_etiqueta`.
4. En `nota.repository.delete`: `prisma.nota.delete({ where: { id } })`.
5. Test integración: 204 éxito, 404 id inexistente, verificar filas relacionadas eliminadas.

**Criterios de done (verificables):**

- [ ] `DELETE /api/v1/notas/:id` responde 204 sin body
- [ ] Nota eliminada no aparece en `GET /notas` ni `GET /notas/:id` (404)
- [ ] Enlaces y `nota_etiqueta` de la nota eliminados (CASCADE)
- [ ] Sin soft-delete (`deleted_at` no existe en MVP)
- [ ] Sin lógica Prisma en controller ni routes

**Tests:**

| Tipo | Ubicación | Qué valida |
|------|-----------|------------|
| Integración | `tests/integration/api/notas.delete.test.ts` | 204, 404, CASCADE relaciones |
| Unitario | `src/backend/src/services/nota.service.test.ts` | 404 si nota no existe |

**Notas técnicas:** `apiClient` debe manejar 204 sin parsear JSON (LLD §7.4). BR-004 en `logical-model-v1.md` §5.
