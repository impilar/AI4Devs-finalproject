## Detalle de implementación (historia)

### Resumen técnico

En `HomePage`, el usuario selecciona una etiqueta en `TagFilter`; el hook `useNotes` vuelve a cargar notas con `GET /api/v1/notas?etiqueta={nombre}`. El backend aplica filtro Prisma `where: { etiquetas: { some: { etiqueta: { name } } } }` usando el índice `idx_nota_etiqueta_etiqueta`. Si no hay resultados, `NoteList` muestra `EmptyState` con mensaje específico. Limpiar filtro restaura listado completo sin query param.

### API involucrada

| Método | Ruta | Uso en esta historia |
|--------|------|----------------------|
| GET | `/api/v1/notas?etiqueta={nombre}` | Listar solo notas con la etiqueta seleccionada |
| GET | `/api/v1/notas` | Listado completo al limpiar filtro |

**Query params:**

| Param | Tipo | Obligatorio | Default | Notas |
|-------|------|-------------|---------|-------|
| `etiqueta` | string | no | — | Nombre exacto (trim); match case-sensitive según BD |
| `sort` | enum | no | `created_at` | `created_at` \| `title` |
| `order` | enum | no | `desc` | `asc` \| `desc` |

**Respuesta 200 con filtro activo:**

```json
{
  "data": [
    {
      "id": "11111111-1111-1111-1111-111111111101",
      "title": "Ideas de proyecto",
      "createdAt": "2026-06-12T10:00:00.000Z",
      "updatedAt": "2026-06-12T10:00:00.000Z"
    }
  ],
  "meta": { "total": 1 }
}
```

**Sin resultados:** `200` con `{ "data": [], "meta": { "total": 0 } }` — no 404 (escenario Gherkin etiqueta "archivo").

## Detalle por task

### TASK-033 — [BE] GET /notas?etiqueta=nombre con filtrado

| Campo | Valor |
|-------|-------|
| **Tipo** | Backend |
| **Release** | MVP |
| **Depende de** | TASK-001 (GET /notas base); TASK-031 (tablas etiquetas M:N) |

**Objetivo:** Filtrar listado de notas por nombre de etiqueta vía query param.

**Archivos a crear o modificar:**

- `src/backend/src/schemas/nota.schema.ts` — `etiqueta` opcional en `ListNotasQuerySchema`
- `src/backend/src/routes/notas.routes.ts` — `validateQuery(ListNotasQuerySchema)` en GET `/`
- `src/backend/src/controllers/nota.controller.ts` — pasar `req.query` a `notaService.list`
- `src/backend/src/services/nota.service.ts` — delegar filtro en repository
- `src/backend/src/repositories/nota.repository.ts` — `findAll({ etiqueta?, sort, order })`

**Pasos de implementación:**

1. Extender `ListNotasParams` con `etiqueta?: string` (LLD §3.1).
2. En `findAll`, si `etiqueta` presente: `where: { etiquetas: { some: { etiqueta: { name: etiqueta } } } }` (LLD §5.4).
3. Mantener `orderBy` por `createdAt` o `title` según query; `select` campos resumen.
4. Controller responde `200 { data: NotaResumen[], meta: { total } }` con `total = data.length` (MVP sin paginación).
5. Etiqueta inexistente o sin notas: `data: []`, `meta.total: 0` (no error).
6. Test integración: seed con notas "trabajo"/"personal"; filtro devuelve subconjunto correcto.

**Criterios de done (verificables):**

- [ ] `GET /api/v1/notas?etiqueta=trabajo` solo notas con etiqueta "trabajo"
- [ ] Sin param `etiqueta`, comportamiento idéntico a US-001
- [ ] `meta.total` coherente con longitud de `data`
- [ ] Query alineada con SQL §9.4 `logical-model-v1.md`

**Tests:**

| Tipo | Ubicación | Qué valida |
|------|-----------|------------|
| Integración | `tests/integration/api/notas.filter-etiqueta.test.ts` | Filtro, vacío, listado completo |

**Notas técnicas:** Match por `etiquetas.name` exacto tras trim en schema. Catálogo de etiquetas con conteo es US-011; el frontend puede derivar lista de tags de notas ya cargadas o seed.
